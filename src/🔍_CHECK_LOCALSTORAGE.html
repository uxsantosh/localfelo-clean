🔍 LocalStorage Session Debugger

Step 1: Open Browser Console (F12)

Then copy and paste this code into the console:

📋 Debug Code (Copy This):

// Check all localStorage keys
console.log('='.repeat(50));
console.log('🔍 LOCALSTORAGE KEYS');
console.log('='.repeat(50));

for (let i = 0; i < localStorage.length; i++) {
    const key = localStorage.key(i);
    console.log(`\n📌 Key: ${key}`);
    
    if (key.includes('sb-') || key.includes('auth') || key.includes('supabase')) {
        const value = localStorage.getItem(key);
        try {
            const parsed = JSON.parse(value);
            console.log('   Value (parsed):', parsed);
            
            if (parsed.access_token) {
                console.log('   ✅ Has access_token:', parsed.access_token.substring(0, 50) + '...');
            }
            if (parsed.refresh_token) {
                console.log('   ✅ Has refresh_token:', parsed.refresh_token.substring(0, 50) + '...');
            }
            if (parsed.expires_at) {
                const expiresAt = new Date(parsed.expires_at * 1000);
                const now = new Date();
                const isExpired = expiresAt < now;
                console.log(`   ${isExpired ? '❌' : '✅'} Expires at:`, expiresAt.toLocaleString());
                console.log(`   ${isExpired ? '❌ EXPIRED!' : '✅ Still valid'}`);
            }
        } catch (e) {
            console.log('   Value (raw):', value);
        }
    }
}

console.log('\n' + '='.repeat(50));
console.log('🔐 SUPABASE SESSION CHECK');
console.log('='.repeat(50));

// Import supabase and check session
import { supabase } from './lib/supabaseClient.ts';

supabase.auth.getSession().then(({ data, error }) => {
    console.log('\n📊 Current Session:', {
        hasSession: !!data.session,
        user: data.session?.user?.id,
        expiresAt: data.session?.expires_at ? new Date(data.session.expires_at * 1000).toLocaleString() : 'N/A',
        error: error
    });
    
    if (!data.session) {
        console.log('\n❌ NO ACTIVE SESSION!');
        console.log('   This is why storage uploads fail.');
        console.log('   Need to login again to create session.');
    } else {
        console.log('\n✅ SESSION IS ACTIVE!');
        console.log('   Storage uploads should work.');
    }
});

console.log('\n' + '='.repeat(50));

What to Look For:

  • ✅ Key starting with "sb-drofnrntrbedtjtpseve-auth-token"
  • ✅ Has "access_token" and "refresh_token"
  • ✅ "expires_at" is in the future (not expired)
  • ❌ If missing = need to login again
  • ❌ If expired = need to refresh or login again

If No Session Found:

The phone OTP login didn't create a Supabase auth session.

This means the verify-otp edge function isn't returning access_token/refresh_token.

Fix: Logout and login again. Watch console for these logs:

🔐 [AuthPhone] About to set session with tokens...
   Access Token: ✅ Present
   Refresh Token: ✅ Present
✅ [AuthPhone] Supabase session set successfully!

If you see "❌ Missing" for tokens, the edge function is broken.
